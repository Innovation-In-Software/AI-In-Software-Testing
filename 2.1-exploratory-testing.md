## Goal

Use AI to systematically generate the categories of nasty input that human testers routinely miss, then execute those inputs against a real API endpoint and analyze the results.

By the end of this lab, you will be able to:

- Use AI to generate boundary values, invalid inputs, and special-character payloads
- Identify validation gaps in a REST API by running AI-generated test cases
- Recognize how injection-style inputs appear in API testing
- Reflect on the difference between what an API does and what it should do

---

## Scenario

You are testing a user registration endpoint. The developer claims all inputs are validated. Your job is to use AI to probe that claim systematically — finding the inputs they did not think to test.

## Starter Code

Create a folder named `lab4-register`, then install dependencies:

```bash
pip install flask pytest
```

Create `app.py`:

```python
from flask import Flask, request, jsonify

app = Flask(__name__)


@app.route('/register', methods=['POST'])
def register():
    data = request.get_json()

    if not data:
        return jsonify({"error": "Request body must be JSON"}), 400

    name = data.get('name')
    email = data.get('email')
    age = data.get('age')

    if not name:
        return jsonify({"error": "name is required"}), 400

    if not email:
        return jsonify({"error": "email is required"}), 400

    if '@' not in email:
        return jsonify({"error": "email must contain @"}), 400

    if age is None:
        return jsonify({"error": "age is required"}), 400

    if not isinstance(age, int):
        return jsonify({"error": "age must be an integer"}), 400

    if age < 18:
        return jsonify({"error": "must be 18 or older to register"}), 400

    return jsonify({
        "message": "Registration successful",
        "user": {"name": name, "email": email, "age": age}
    }), 201


if __name__ == '__main__':
    app.run(debug=True)
```

Create `test_register.py`:

```python
import pytest
import json
from app import app


@pytest.fixture
def client():

        '/register',
        data=json.dumps(payload),
        content_type='application/json'
    )
```

Verify setup: `pytest test_register.py -v` should collect zero tests with no errors.

## API Weaknesses Revealed

> **Instructor Note:** Share after students complete all parts.

| Input | What Happens | Why It Is a Problem |
|---|---|---|
| `name = "   "` | Returns 201 | `if not name:` is truthy for whitespace. Fix: `if not name.strip():` |
| `age = 999` | Returns 201 | No upper bound on age. Fix: add `if age > 120:` |
| `age = true` (JSON) | Returns 400 — but for the wrong reason | Python: `bool` is a subclass of `int`. `True < 18` is `True`. Fix: `if isinstance(age, bool) or not isinstance(age, int):` |
| `email = "alice@"` | Returns 201 | Only checks `'@' in email`. Fix: use a regex or validation library. |
| Name of 10,000 chars | Returns 201 | No length limit. In production: database truncation or DoS risk. |
| SQL / XSS strings | Returns 201, echoed back | No sanitization. In a system with a DB or HTML output, these would be exploitable. |

## Key Takeaways

Human exploratory testing clusters around the happy path. AI generates inputs by category — boundaries, type confusion, oversized payloads, injection patterns — systematically covering ground humans find tedious or unintuitive.

AI generates the inputs. The tester decides which results are bugs.

The most dangerous result is not a crash — it is a silent 201 on input that should have been rejected.

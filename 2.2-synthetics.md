## Starter Code

Create a folder named `lab5-ecommerce` and install dependencies:

```bash
sudo dnf install -y sqlite
pip install fastapi uvicorn faker
```

Create `schema.sql`:

```sql
CREATE TABLE IF NOT EXISTS customers (
    id      INTEGER PRIMARY KEY AUTOINCREMENT,
    name    TEXT NOT NULL,
    email   TEXT UNIQUE NOT NULL,
    city    TEXT,
    country TEXT,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS products (
    id       INTEGER PRIMARY KEY AUTOINCREMENT,
    name     TEXT NOT NULL,
    category TEXT,
    price    REAL NOT NULL,
    stock    INTEGER DEFAULT 100
);

CREATE TABLE IF NOT EXISTS orders (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    customer_id INTEGER NOT NULL,
    product_id  INTEGER NOT NULL,
    quantity    INTEGER DEFAULT 1,
    total       REAL NOT NULL,
    status      TEXT DEFAULT 'pending',
    created_at  TEXT DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (customer_id) REFERENCES customers(id),
    FOREIGN KEY (product_id)  REFERENCES products(id)
);
```

Create `app.py`:

```python
import sqlite3
from fastapi import FastAPI, HTTPException, Depends
from fastapi.responses import RedirectResponse
from pydantic import BaseModel

app = FastAPI()
DATABASE = 'ecommerce.db'


@app.get("/")
def root():
    return RedirectResponse(url="/docs")


def get_db():
    db = sqlite3.connect(DATABASE)
    db.row_factory = sqlite3.Row
    try:
        yield db
    finally:
        db.close()


@app.get("/products")
def list_products(db: sqlite3.Connection = Depends(get_db)):
    products = db.execute('SELECT * FROM products LIMIT 50').fetchall()
    return [dict(p) for p in products]


@app.get("/customers/{customer_id}/orders")
def customer_orders(customer_id: int, db: sqlite3.Connection = Depends(get_db)):
    rows = db.execute(
        '''SELECT o.id, o.quantity, o.total, o.status, p.name AS product_name
           FROM orders o JOIN products p ON o.product_id = p.id
           WHERE o.customer_id = ?''',
        (customer_id,)
    ).fetchall()
    if not rows:
        raise HTTPException(status_code=404, detail="no orders found")
    return [dict(r) for r in rows]


class OrderRequest(BaseModel):
    customer_id: int
    product_id: int
    quantity: int = 1


@app.post("/orders", status_code=201)
def place_order(order: OrderRequest, db: sqlite3.Connection = Depends(get_db)):
    product = db.execute('SELECT * FROM products WHERE id = ?', (order.product_id,)).fetchone()
    if not product:
        raise HTTPException(status_code=404, detail="product not found")

    total = product['price'] * order.quantity
    db.execute(
        'INSERT INTO orders (customer_id, product_id, quantity, total) VALUES (?, ?, ?, ?)',
        (order.customer_id, order.product_id, order.quantity, total)
    )
    db.commit()
    return {"message": "order placed", "total": total}
```

Initialize the database schema:

```bash
sqlite3 ecommerce.db < schema.sql
```

Start the API (leave this terminal open for the rest of the lab):

```bash
uvicorn app:app --reload --host 0.0.0.0
```

Open a new terminal while keeping the other one running.

Confirm it works: `curl http://localhost:8000/products` should return an empty list.

FastAPI also generates interactive API docs automatically. Open `http://localhost:8000/docs` in a browser to explore and test the endpoints with a UI.

## Key Takeaways

AI generates data quickly, but quantity is not the same as quality. Ten thousand rows of poorly distributed data is worse than one hundred well-constructed rows — it creates false confidence.

The validation step is not optional. AI models produce plausible-looking data that can still be wrong in systematic ways: all customers from one country, prices in a single tight range, emails that follow one obvious pattern.

**Validate before you load. Realistic data produces meaningful tests. Unrealistic data produces misleading ones.**

# Day 1 Lab: AI-Assisted Test Creation & Coverage

## Lab 1: Manual Testing vs. AI-Assisted Testing

**Duration:** 45–60 minutes  
**Audience:** Software testers, QA professionals, and developers  
**Tools:** Python, pytest, ChatGPT or GitHub Copilot

---

## Goal

Demonstrate the value and limitations of AI-assisted test generation by comparing manually written tests against tests generated with an AI tool.

By the end of this lab, you will be able to:

- Write unit tests manually for a small Python function
- Identify common edge cases in business logic
- Use ChatGPT or GitHub Copilot to generate unit tests
- Compare manual and AI-generated test coverage
- Discuss where AI helps and where human review is still required

---

## Scenario

You work on a small e-commerce application. The application applies customer discounts during checkout.

The business rules are simple at first glance, but there are several edge cases that testers often miss.

Your task is to test the `calculate_discounted_price()` function.

---

## Starter Code

Create a file named `discount.py`:

```python
def calculate_discounted_price(price, customer_type, coupon_code=None, is_holiday=False):
    """
    Calculate the final discounted price for a customer.

    Business rules:
    - premium customers receive 20% off
    - standard customers receive 10% off
    - guests receive no customer discount
    - coupon code SAVE10 gives an additional 10% off
    - coupon code SAVE20 gives an additional 20% off, but only for premium customers
    - holiday promotion gives an additional 5% off
    - discounts are applied sequentially
    - final price is rounded to 2 decimal places
    - price must be greater than or equal to 0
    - unknown customer types are treated as guests
    - unknown coupon codes are ignored
    """

    if price < 0:
        raise ValueError("price cannot be negative")

    customer_type = customer_type.lower()

    if customer_type == "premium":
        price = price * 0.8
    elif customer_type == "standard":
        price = price * 0.9

    if coupon_code == "SAVE10":
        price = price * 0.9
    elif coupon_code == "SAVE20" and customer_type == "premium":
        price = price * 0.8

    if is_holiday:
        price = price * 0.95

    return round(price, 2)
```

---

## Why This Function Is Better Than a Basic Example

A simpler discount function usually only checks customer type. That is useful for introducing unit tests, but it does not create enough opportunities for missed cases.

This version includes several realistic testing concerns:

- Multiple input combinations
- Sequential discounts
- Invalid input
- Case sensitivity
- Unknown customer types
- Unknown coupon codes
- Rounding behavior
- Conditional coupon eligibility

These details make it easier to compare human-created tests against AI-generated tests.

## Part 1: Manual Test Creation

Create a file named `test_discount_manual.py`.

Write your own pytest tests before using AI.

Example starter test:

```python
from discount import calculate_discounted_price


def test_premium_customer_gets_20_percent_discount():
    result = calculate_discounted_price(100, "premium")
    assert result == 80.00
```

### Manual Testing Tasks

Write tests for the function manually.

Do not use ChatGPT, Copilot, or another AI tool yet.

Try to answer these questions as you test:

1. What are the most obvious test cases?
2. What edge cases might break the function?
3. What combinations of inputs matter?
4. What invalid inputs should be tested?
5. What behavior needs to be confirmed from the business rules?

## Part 3: Use AI to Generate Tests

Now use ChatGPT, Claude, GitHub Copilot, or another AI tool to generate tests.

Create a file named `test_discount_ai.py`.

### Suggested Zero-Shot Prompt

Use this prompt first:

```text
Write pytest unit tests for the following Python function. Include normal cases and edge cases.

[paste the calculate_discounted_price function here]
```

Run the AI-generated tests:

```bash

```

## Part 4: Improve the Prompt

Now try a more specific prompt. You can also use other AI systems to help you build a prompt.

```text
Write a complete pytest test suite for this function.

Focus on:
- customer types: premium, standard, guest, and unknown values
- valid coupons: SAVE10 and SAVE20
- invalid coupon codes
- holiday and non-holiday pricing
- negative prices
- zero price
- rounding behavior
- combinations where multiple discounts apply
- cases where SAVE20 should not apply

Also include clear test names.

[paste the function here]
```

Save the improved AI tests in `test_discount_ai_improved.py`.

Run them:

```bash
pytest test_discount_ai_improved.py -v
```

## Common Missed Cases

Students often miss some of these cases:

- `SAVE20` only works for premium customers
- Unknown customer types are treated as guests
- Unknown coupon codes are ignored
- Discounts are applied sequentially, not added together
- Holiday discount applies after other discounts
- Negative prices should raise an error
- Zero price is allowed
- Rounding can change the final result
- Customer type is case-insensitive

---

## Key Takeaways

AI can quickly generate a broad set of unit tests, but it is not a replacement for understanding requirements.

Good testers still need to:

- Read the business rules carefully
- Identify missing or ambiguous requirements
- Review AI-generated tests for correctness
- Improve prompts when initial results are weak
- Validate that tests actually check meaningful behavior

AI is most useful when treated as a testing assistant, not as the final authority.
